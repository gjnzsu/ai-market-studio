## Why

AI Market Studio now has a single workflow/playbook chat runtime, but its market analysis workflow still returns a generic `market_analysis` payload with limited FX-specific structure. A first-version FX analysis agent needs a stable, source-grounded analysis contract that can become the baseline for later playbooks and autonomy experiments.

## What Changes

- Add a workflow-first FX analysis capability for single-pair and multi-pair FX questions.
- Enhance `analyze_market_context` so FX analysis requests can return a structured `fx_analysis` payload.
- Keep the model-facing tool surface unchanged: `collect_market_context`, `analyze_market_context`, and `generate_market_briefing`.
- Add deterministic first-version analysis sections:
  - market context summary
  - trend and momentum
  - volatility snapshot
  - macro/news/research drivers when sources are available
  - bull/base/bear scenarios
  - watch items, data gaps, confidence, and research-only framing
- Preserve `generate_market_briefing` as the path for full narrative reports and playbook-style briefings.
- Do not add autonomous planning in this change.

## Capabilities

### New Capabilities

- `fx-analysis-workflow`: Structured, research-only FX analysis output generated by workflow internals.

### Modified Capabilities

- `intent-level-agent-workflows`: Clarify that `analyze_market_context` can return an FX-specific analysis payload for FX pair analysis intents.

## Impact

- Backend workflows: `backend/agent/workflows.py`
- Analysis internals: likely `backend/agents/market_analyst.py` or a new internal specialist module if needed
- Tool schema: `analyze_market_context` may gain optional parameters such as `horizon` and `focus`
- Tests: unit tests for FX analysis shape, source grounding, data gaps, and research-only guardrails; agent-loop tests for tool selection
- Docs: README section describing the first-version FX analysis workflow
