Workflow Examples

Let's validate the design with real use cases

You've chosen Sub-Agents as Function Tools (A) — each sub-agent is a Python function that GPT-4o/5.x calls via function calling.

Let's walk through some example workflows to make sure this architecture handles your use cases. Select the workflow you'd like to explore first:

A

Simple Query: "What is EUR/USD today?"

Expected flow:

  1. User sends message
  2. Orchestrator (GPT-4o) detects intent: spot rate query
  3. Calls collect_market_data(pairs=["EUR/USD"], type="spot")
  4. Data Collector fetches from ExchangeRateHostConnector
  5. Returns rate to orchestrator
  6. Orchestrator composes natural language response

Sub-agents involved: Data Collector only

Parallel execution: No (single tool call)

B

Multi-Step: "Analyze EUR/USD trend and create a PDF report"

Expected flow:

  1. User sends message
  2. Orchestrator detects intent: trend analysis + report
  3. Calls collect_market_data(pairs=["EUR/USD"], days=30)
  4. Data Collector fetches historical rates
  5. Calls analyze_market_trends(data=rates)
  6. Market Analyst computes trends, volatility, signals
  7. Calls generate_report(analysis=results, format="pdf")
  8. Report Generator creates PDF
  9. Returns PDF download link

Sub-agents involved: Data Collector → Market Analyst → Report Generator

Parallel execution: No (sequential pipeline)

C

Parallel: "Give me market insight on EUR/USD, GBP/USD, and USD/JPY"

Expected flow:

  1. User sends message
  2. Orchestrator detects intent: multi-pair market insight
  3. Parallel calls:
    • collect_market_data(pairs=["EUR/USD","GBP/USD","USD/JPY"])
    • collect_market_data(type="news", query="FX")
    • collect_market_data(type="fred", series=["DFF"])
  4. Calls synthesize_research(rates=..., news=..., indicators=...)
  5. Research Synthesizer combines all sources
  6. Returns inline insight with rate chips + news cards

Sub-agents involved: Data Collector (3x parallel) → Research Synthesizer

Parallel execution: Yes (3 data collection calls in parallel)

D

Complex: "Research EUR/USD outlook, analyze trend, and generate dashboard"

Expected flow:

  1. User sends message
  2. Orchestrator detects intent: research + analysis + visualization
  3. Parallel calls:
    • collect_market_data(pairs=["EUR/USD"], days=30)
    • collect_market_data(type="rag", query="EUR/USD outlook")
    • collect_market_data(type="news", query="EUR USD")
  4. Calls synthesize_research(rates=..., rag=..., news=...)
  5. Calls analyze_market_trends(data=rates, context=synthesis)
  6. Calls generate_report(analysis=..., format="dashboard")
  7. Returns inline Chart.js dashboard

Sub-agents involved: Data Collector (3x parallel) → Research Synthesizer → Market Analyst → Report Generator

Parallel execution: Yes (initial data collection), then sequential

Key insight: GPT-4o/5.x function calling already supports parallel tool calls when they're independent. Your orchestrator will automatically parallelize Data Collector calls when fetching from multiple sources.
