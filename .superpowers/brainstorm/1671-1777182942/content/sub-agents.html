Sub-Agent Responsibilities

What specialized agents should we create?

Now that we're using a Hybrid (Hierarchical + Parallel) architecture, let's define the sub-agents. Based on your existing features, here are the natural boundaries:

Proposed Agent Team

Select the agents you want to include in your system. You can pick multiple.

A

Data Collector Agent

Responsibility: Fetch raw market data from all sources

Tools: ExchangeRateHostConnector, FREDConnector, RSSNewsConnector, RAGConnector

Input: Data request (pairs, date ranges, series IDs, queries)

Output: Raw structured data (rates, news, economic indicators, research)

This agent knows HOW to fetch data but doesn't analyze it.

B

Market Analyst Agent

Responsibility: Analyze trends, correlations, patterns in market data

Tools: Statistical analysis, trend detection, correlation analysis

Input: Raw market data (from Data Collector)

Output: Analysis results (trends, insights, signals, risk assessments)

This agent understands WHAT the data means.

C

Report Generator Agent

Responsibility: Create formatted outputs (PDFs, dashboards, summaries)

Tools: PDF skill, Chart.js rendering, dashboard builder

Input: Analysis results + raw data

Output: Formatted reports, visualizations, export files

This agent knows HOW to present information.

D

Risk Monitor Agent

Responsibility: Watch for anomalies, threshold breaches, risk events

Tools: Threshold detection, anomaly detection, alert generation

Input: Real-time data stream + risk parameters

Output: Alerts, warnings, risk scores

This agent watches for WHEN things go wrong.

E

Research Synthesizer Agent

Responsibility: Combine multi-source intelligence into coherent insights

Tools: RAG queries, news analysis, economic indicator correlation

Input: Research docs + news + economic data

Output: Synthesized intelligence reports, context-aware insights

This agent connects the dots across sources.

F

Conversation Manager Agent

Responsibility: Handle user interaction, maintain context, route requests

Tools: Chat history, user preferences, intent detection

Input: User messages, conversation history

Output: Routed requests to other agents, conversational responses

This agent understands WHAT the user wants.

Recommendation: Start with A (Data Collector), B (Market Analyst), and C (Report Generator) as your core trio. Add D (Risk Monitor) and E (Research Synthesizer) for advanced features. F (Conversation Manager) can be your orchestrator itself.
