## 1. Project Overview

**Goal:** Build a Proof-of-Concept (PoC) layered FX market data chatbot that allows users to query foreign exchange rates via a natural-language chat interface.

**Purpose:** Demonstrate a clean, extensible architecture for AI-powered market data retrieval. The PoC prioritises clarity of design over production hardening — it should be easy to hand off, extend, and understand.

**Tech Stack:**

| Layer | Technology |
|---|---|
| Frontend | Plain HTML + Vanilla JS (single file, no build step) |
| Backend API | Python 3.11+ + FastAPI |
| AI Agent Layer | OpenAI GPT-4o with function calling (no agent framework) |
| Data Connector Layer | Python abstract base class + exchangerate.host adapter |
| Market Data Source | exchangerate.host REST API (free tier, no API key required) |
| Testing | pytest (unit + agent tests), httpx TestClient (E2E) |

## 2. Layered Architecture

```
[Chat UI] -> [Backend API /api/chat] -> [AI Agent Layer] -> [Data Connector Layer] -> [Market Data Sources]
```

### Data Flow (single request)

1. User types a message in the chat UI (e.g. "What is EUR/USD right now?").
2. JS sends POST /api/chat with { messages: [...] } to FastAPI.
3. FastAPI passes the conversation to the AI Agent Layer.
4. The agent calls openai.chat.completions.create with tool definitions.
5. GPT-4o responds with a tool_call (e.g. get_exchange_rate(base="EUR", target="USD")).
6. The agent layer invokes the Data Connector with the extracted arguments.
7. The connector calls https://api.exchangerate.host/live (or equivalent endpoint).
8. The connector returns structured data back to the agent.
9. The agent sends the tool result back to GPT-4o for final response synthesis.
10. GPT-4o returns a natural-language reply.
11. FastAPI returns { reply: "..." } to the browser.
12. JS renders the reply in the chat window.

## 4. Data Connector Layer — Design Rationale

### Why an abstract base class?

The PoC uses exchangerate.host because it is free and requires no API key. In a real product the source would be Bloomberg, Refinitiv, or an internal feed. By defining an abstract interface now, swapping the source requires only adding a new file under `connectors/` and updating `config.py`. No changes to the agent or API layer are needed.

### Abstract Interface

```python
# backend/connectors/base.py
from abc import ABC, abstractmethod
from typing import Optional

class MarketDataConnector(ABC):

    @abstractmethod
    async def get_exchange_rate(
        self,
        base: str,
        target: str,
        date: Optional[str] = None,  # ISO 8601, None = latest
    ) -> dict:
        # Returns: {"base": "EUR", "target": "USD", "rate": 1.0823,
        #           "date": "2026-03-25", "source": "exchangerate.host"}
        ...

    @abstractmethod
    async def list_supported_currencies(self) -> list[str]:
        """Returns list of supported ISO 4217 currency codes."""
        ...
```


### ExchangeRateHost Adapter

- Uses `httpx` for async HTTP calls.
- Endpoints:
  - Latest: `https://api.exchangerate.host/live?base={base}&symbols={target}`
  - Historical: `https://api.exchangerate.host/historical?date={date}&base={base}&symbols={target}`
  - Currencies: `https://api.exchangerate.host/list`
- All responses normalised to the dict schema above before returning to the agent.

## 5. AI Agent Layer — Design

### Approach

No agent framework (no LangChain, no AutoGen). The function-calling loop is implemented directly against the OpenAI Python SDK. This keeps the PoC transparent and dependency-light.

### Function-Calling Loop (pseudo-code)

```python
while True:
    response = await openai_client.chat.completions.create(
        model="gpt-4o",
        messages=conversation_history,
        tools=TOOL_DEFINITIONS,
        tool_choice="auto"
    )
    message = response.choices[0].message
    if message.tool_calls:
        for tool_call in message.tool_calls:
            result = await dispatch_tool(tool_call, connector)
            conversation_history.append(tool_result_message(tool_call.id, result))
    else:
        return message.content  # final reply
```

### Tool Definitions (two tools for PoC)

**get_exchange_rate** — parameters: `base` (str, required), `target` (str, required), `date` (str, optional ISO 8601)

**list_supported_currencies** — no parameters

Both are defined as JSON schema objects in `backend/agent/tools.py` and passed as the `tools` array to the OpenAI API.

### System Prompt

```
You are an FX market data assistant. Help users query foreign exchange rates.
Always use the provided tools to fetch live or historical data — never guess rates.
Be concise and professional. Format rates to 4 decimal places.
```

### Conversation History Management

- Stateless PoC: the frontend sends the full `messages` array with each request.
- The backend prepends the system prompt, runs the agent loop, and returns the updated messages plus the final reply.
- No server-side session storage is required.

## 8. Testing Strategy

### Unit Tests (`tests/unit/`)

- `test_connector.py`: Tests `ExchangeRateHostConnector` with mocked `httpx` responses (use `respx`). Covers URL construction, response parsing, error handling.
- `test_tools.py`: Validates tool JSON schemas are well-formed and tool dispatch resolves correctly.
- Tools: `pytest`, `pytest-asyncio`, `respx`

### Agent Tests (`tests/agent/`)

- `test_agent.py`: Tests the function-calling loop with a fully mocked `openai.AsyncOpenAI` client.
- Covers: correct tool invocation, final reply extraction, multi-turn sequences, connector error handling.
- Tools: `pytest`, `pytest-asyncio`, `unittest.mock`

### E2E Tests (`tests/e2e/`)

- `test_chat_api.py`: Uses FastAPI `TestClient` (httpx) to test the full HTTP stack.
- Mocks both OpenAI client and exchangerate.host HTTP calls — no live network required.
- Covers: valid request/response structure, malformed input returns HTTP 422, missing API key returns HTTP 500.
- Tools: `pytest`, `httpx`, `respx`, `unittest.mock`

## 10. Key Decisions & Constraints

1. **No agent framework** — GPT-4o function calling is wired directly. No LangChain, AutoGen, or similar.
2. **No API key for data source** — exchangerate.host free tier requires no authentication.
3. **Stateless backend** — conversation history lives in the browser; backend is purely request/response.
4. **Single HTML file frontend** — no build tooling, no npm, no TypeScript.
5. **Async throughout** — all connector and agent code uses `async/await`; FastAPI runs async endpoints.
6. **Abstract connector interface** — must be preserved so future data sources can be plugged in without touching agent or API code.
7. **No production hardening** — no auth, no rate limiting, no persistence. PoC only.
8. **Python 3.11+** — use modern type hints (`list[str]` not `List[str]`, `dict` not `Dict`).
